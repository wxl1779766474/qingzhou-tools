# Android WebUSB 性能测试使用指南

## 使用条件

- 桌面版 Chrome 或 Edge，页面必须通过 HTTPS 或 `localhost` 打开。
- Android 8 或更高版本；Android 10 及以上通常能提供更完整的热状态信息。
- 一根支持数据传输的 USB 线。只支持充电的线缆无法连接。
- 手机已开启“开发者选项”和“USB 调试”。

Safari、Firefox、手机浏览器、无线 ADB、模拟器和 iOS 暂不支持。

## 开始测试

1. 解锁手机并连接电脑。必要时把 USB 用途切换为“传输文件”。
2. 打开“Android 性能测试”，点击“连接 Android 设备”。
3. 在浏览器设备选择器中选中手机，再在手机的 RSA 调试授权弹窗中点击“允许”。
4. 页面会自动读取设备信息并识别当前前台 App。也可以搜索第三方应用包名，或手动输入类似 `com.example.app` 的标准包名。
5. 选择测试时长并点击“开始测试”。测试期间在手机上正常操作目标 App。
6. 点击“停止并生成报告”。设备断开、目标进程长时间退出或达到设定时长时也会自动生成部分或完整报告。

单场测试最长 60 分钟。测试运行时切换其他工具会先提示停止并收尾。

## 指标说明

- **应用 CPU**：目标包主进程和子进程的 CPU 合计，归一化为整机占比 0–100%。设备命令不兼容时会降级采集。
- **PSS 内存**：目标全部进程的 `TOTAL PSS` 合计，并尽量保留 Java Heap、Native Heap 和 RSS。
- **活动渲染 FPS 与卡顿率**：来自普通 View/HWUI 的帧统计，只在采样窗口内有新帧时估算。静止画面显示“—”，不是 0 FPS。
- **网络区间流量**：通过目标 UID 在测试开始和结束读取快照，展示 RX/TX 增量，不是持续实时网速。
- **设备电池温度**：来自电池传感器，不是 CPU/GPU 温度，也不代表 App 精确功耗。
- **热状态**：Android 10+ 且厂商实现允许时提供；缺失不会影响其他指标。

Unity、Unreal、OpenGL 或 Vulkan 游戏可能没有可靠的 HWUI 帧数据。VPN、系统下载服务、代理和厂商实现可能影响网络 UID 统计。USB 通常处于充电状态，短测不能根据电量百分比推断应用耗电。

## 常见问题

### 设备没有出现在选择器中

- 确认使用桌面 Chrome/Edge 和 HTTPS 页面。
- 保持手机解锁，检查 USB 调试是否开启。
- 切换 USB 用途为“传输文件”。
- 更换确认支持数据传输的线缆或 USB 端口。

### 提示 USB 接口被占用

Android Studio、本机 `adb` 或其他调试工具可能已认领接口。关闭相关程序；必要时在终端运行 `adb kill-server`，重新插拔手机后再连接。网页不会在电脑上代为执行任何命令。

### 想重置网页调试授权

浏览器 ADB RSA 私钥保存在当前站点来源的浏览器数据中。请在浏览器站点设置中清除本站数据，同时可在手机“撤销 USB 调试授权”后重新连接。

## 报告与隐私

- 设备信息、采样值和报告不会上传服务器，也不会写入 URL 或分析事件。
- 页面不展示或导出完整设备序列号，只保留掩码。
- 最近 20 份报告保存在当前浏览器 IndexedDB；点击报告只恢复查看，不会重新连接或执行命令。
- 可导出版本化 JSON 或 CSV。浏览器存储不可用时，当前报告仍可立即导出。
- 这是一项快速诊断工具，不替代 Perfetto、实验室功耗设备或方法级性能分析。
